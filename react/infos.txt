- Como ver a versão do node.js:
    1. abra o cmd
    2. digite "node -v"

- Como ver a versão do npm:
    1. abra o cmd
    2. digite "npm -v"

- Como criar um ambiente usando Vite:
    1. abra o cmd
    2. digite "npm create vite@latest"
    3. adicionar o nome do projeto
    4. adicionar ambiente
    5. adicionar linguagem

- Como rodar o ambiente:
    1. digite "npm run dev"

- Estrutura do projeto:
    1. node_modules: dependências do projeto
    2. public: assets e arquivos estáticos
    3. src: onde fica o código da aplicação
    4. src/index.js: arquivo de inicialização da aplicação
    5. src/App.js: componente principal inicial

- Comentários em jsx:
    1. //comentario
    2. {/* comentario */}

- Ao criar um novo componente:
    1. Para não precisar criar a arrow function, exportar, digite: rafce + tab
    2. Antes do return é js puro, depois do return é jsx
    3. O que estiver entre chaves "{}" é javascript

- Hooks:
    1. Hooks em React são funções especiais que permitem usar recursos do React (como estado e efeitos) dentro de componentes funcionais, sem precisar usar classes. 
    2. "se o valor muda e precisa aparecer na tela, usa o state"

- Props:
    1. Permite a passagem de dados de um componente pai para um componente filho
    2. Útil para quando houver dados vindos de um banco de dados
    3. Normalmente, em componentes, há mais de um Prop
    4. Para facilitar o uso dos props, é comum desestruturas no parâmetro da função do componente

- Map:
    1. Invoca a função callback passada por argumento para cada elemento do array e devolve um novo array como resultado
    2. Estrutura básica: array.map((item) => item * 3);

- Children:
    1. É uma prop especial do React que representa tudo o que você coloca entre a abertura e o fechamento de um componente

- State lift:
    1. Leva um valor do componente filho pro componente pai

- CSS de componente:
    1. É utilizado apenas em um componente em específico
    2. não é scoped, ou seja, pode "vazar" para outros elementos do projeto

- CSS Modules:
    1. Permite deixar o css scoped, funcionando apenas para um componente em questao
